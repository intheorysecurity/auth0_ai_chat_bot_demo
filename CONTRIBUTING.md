# Contributing

Thanks for helping improve this project.

## Getting started

1. Fork the repository and clone your fork.
2. Follow [docs/DEVELOPMENT.md](docs/DEVELOPMENT.md) for local setup.
3. Create a branch: `git checkout -b feat/your-topic` or `fix/issue-description`.

## Pull requests

- Keep changes focused; prefer small PRs.
- Update docs (`README.md`, `docs/`) if behavior or configuration changes.
- Do **not** commit secrets, real `.env` files, or production credentials.
- Add screenshots to `docs/images/` only if they contain no sensitive data (or use placeholders).

Use the PR template when opening a request on GitHub.

## Style

- **Python:** PEP 8–aligned formatting; type hints where practical.
- **TypeScript/React:** Follow existing patterns in `frontend/src`; run `npx tsc --noEmit` before pushing.

## Questions

Use your repository’s **Issues** or **Discussions** (if enabled) for design questions before large refactors.
